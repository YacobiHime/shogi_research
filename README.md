# 将棋エンジンのUSIオプション最適化

やねうら王のUSIオプションをCMA-ESで自動調整し、ランダムサーチと比較する小規模な研究です。

実験は、Google Colab上の1つのノートブックで実行します。

## 研究目的

以下を検証します。

1. USIオプションの調整によって、デフォルト設定より高い対局成績を得られるか
2. 同じ候補数と対局数において、CMA-ESはランダムサーチより良い設定を発見できるか

本研究では、人手による調整との比較は行いません。

## 最適化対象

やねうら王がUSIオプションとして公開している設定を対象とします。

候補は以下のとおりです。

| オプション                   | 内容              |
| ----------------------- | --------------- |
| `BookEvalDiff`          | 定跡候補として許容する評価値差 |
| `BookDepthLimit`        | 定跡を利用する深さに関する設定 |
| `ConsiderBookMoveCount` | 定跡の採択数を考慮するか    |

オプション名や利用可能範囲は、やねうら王のバージョンによって異なる可能性があります。

そのため、実験開始時にUSIオプション一覧を取得し、実際に存在するものだけを使用します。

## 比較手法

以下を比較します。

* デフォルト設定
* CMA-ES
* ランダムサーチ

CMA-ESとランダムサーチでは、以下を同じ条件にします。

* 評価候補数
* 候補当たりの対局数
* 総対局数
* 開始局面
* 先手・後手の割り当て

## 評価方法

候補設定のエンジンと、デフォルト設定のエンジンを対局させます。

勝ちを1点、引き分けを0.5点、負けを0点として平均得点を計算します。

[
S(x)=
\frac{
N_{\mathrm{win}}+0.5N_{\mathrm{draw}}
}{
N_{\mathrm{games}}
}
]

同じ開始局面について先手と後手を入れ替え、2局を1組として評価します。

## 小規模実験の予定

暫定的に、以下の規模で実施します。

| 項目        | CMA-ES | ランダムサーチ |
| --------- | -----: | ------: |
| 評価候補数     |     30 |      30 |
| 候補当たりの対局数 |    10局 |     10局 |
| 総対局数      |   300局 |    300局 |

CMA-ESでは、1世代当たり6候補、5世代を目安とします。

探索後、それぞれの手法で最も成績が良かった候補を追加で50局程度評価します。

対局数はColabの実行時間に応じて調整します。

## 実験の流れ

1. 必要なライブラリをインストールする
2. やねうら王を準備する
3. USIオプション一覧を確認する
4. 対象オプションと探索範囲を設定する
5. 開始局面を準備する
6. CMA-ESを実行する
7. ランダムサーチを実行する
8. 各手法の最良候補を再評価する
9. 結果を表とグラフで比較する

## 注意点

候補当たり10局程度では、結果に大きなばらつきがあります。

そのため、探索中の最高得点だけで優劣を決めず、各手法の最良候補を追加対局によって再評価します。

本研究は小規模な検証であるため、結果は使用したエンジン、定跡、開始局面、対局条件に限定して解釈します。
